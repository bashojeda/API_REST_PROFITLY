# Profitly REST API

A Node.js Express API for the Profitly financial tracker app, migrating all business logic from Android to backend.

## 🚀 Deployment on Railway

This API is configured for easy deployment on Railway (free MySQL included).

### Prerequisites
- GitHub account
- Railway account (free)

### 1. Push to GitHub
```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/profitly-api.git
git push -u origin main
```

### 2. Deploy on Railway
1. Go to [Railway.app](https://railway.app)
2. Click "New Project" → "Deploy from GitHub"
3. Connect your GitHub repo
4. Railway will auto-detect Node.js and MySQL
5. Add environment variables in Railway dashboard:
   - `DB_HOST` = (auto-generated by Railway)
   - `DB_USER` = (auto-generated by Railway)
   - `DB_PASSWORD` = (auto-generated by Railway)
   - `DB_NAME` = (auto-generated by Railway)
   - `PORT` = 3000

### 3. Get Your API URL
After deployment, Railway gives you a URL like:
```
https://profitly-api-production.up.railway.app
```

### 4. Update Android App
In your Android `ApiConfig.kt`:
```kotlin
const val BASE_URL = "https://profitly-api-production.up.railway.app/api/"
```

### 5. Test Your Deployed API
```bash
curl https://your-railway-url.up.railway.app/health
curl https://your-railway-url.up.railway.app/api/analytics/summary
```

## Features

- RESTful API for managing product sales and expenses
- Analytics endpoints for financial summaries, charts, and insights
- MySQL database with connection pooling
- Input validation and error handling
- CORS enabled for Android app integration

## Setup Instructions

### Prerequisites

- Node.js (v14 or higher)
- MySQL Server
- npm or yarn

### 1. Install Dependencies

```bash
npm install
```

### 2. Set Up MySQL Database

1. Start your MySQL server
2. Create a database and run the schema:

```bash
mysql -u root -p < database.sql
```

This will create the `profitly_db` database with the required tables.

### 3. Configure Database Connection

Update the database credentials in `config/db.js` or set environment variables:

```bash
export DB_HOST=localhost
export DB_USER=root
export DB_PASSWORD=your_password
export DB_NAME=profitly_db
```

### 4. Run the Server

For development:
```bash
npm run dev
```

For production:
```bash
npm start
```

The server will start on port 3000 (or PORT environment variable).

## API Endpoints

Base URL: `http://localhost:3000/api`

### Sales

- **GET /sales** - Get all product sales
- **POST /sales** - Create a new product sale

### Expenses

- **GET /expenses** - Get all expenses
- **POST /expenses** - Create a new expense

### Analytics

- **GET /analytics/summary** - Get financial summary
- **GET /analytics/charts** - Get sales and profit charts over time
- **GET /analytics/insights** - Get business insights

## Example API Requests

### Create a Product Sale

```bash
curl -X POST http://localhost:3000/api/sales \
  -H "Content-Type: application/json" \
  -d '{
    "productName": "Widget A",
    "sellingPrice": 25.50,
    "productionCost": 15.00,
    "quantitySold": 10,
    "createdAtMillis": 1640995200000
  }'
```

### Create an Expense

```bash
curl -X POST http://localhost:3000/api/expenses \
  -H "Content-Type: application/json" \
  -d '{
    "description": "Office supplies",
    "amount": 150.00,
    "createdAtMillis": 1640995200000
  }'
```

### Get Financial Summary

```bash
curl -X GET http://localhost:3000/api/analytics/summary
```

Response:
```json
{
  "totalRevenue": 255.00,
  "totalCosts": 300.00,
  "netProfit": -45.00,
  "profitMarginPercent": -17.65
}
```

### Get Charts Data

```bash
curl -X GET http://localhost:3000/api/analytics/charts
```

Response:
```json
{
  "salesOverTime": [
    {
      "label": "01/01",
      "value": 255.00
    }
  ],
  "profitOverTime": [
    {
      "label": "01/01",
      "value": -45.00
    }
  ]
}
```

### Get Insights

```bash
curl -X GET http://localhost:3000/api/analytics/insights
```

Response:
```json
{
  "insights": [
    "Your costs are too high...",
    "Your profit margin is low. Consider increasing prices or reducing costs."
  ]
}
```

## Business Logic

- **Revenue**: SUM(sellingPrice × quantitySold)
- **Production Cost**: SUM(productionCost × quantitySold)
- **Total Costs**: Production Cost + SUM(expenses)
- **Net Profit**: Revenue - Total Costs
- **Profit Margin**: (Net Profit / Revenue) × 100 (handles division by zero)

Charts group data by day using `createdAtMillis`, with labels formatted as "MM/dd".

## Error Handling

The API returns appropriate HTTP status codes:
- 200: Success
- 201: Created
- 400: Bad Request (validation errors)
- 404: Not Found
- 500: Internal Server Error

Validation errors are returned as:
```json
{
  "errors": ["field is required", "value must be positive"]
}
```

## Connecting to Android App

This API is designed to work with Retrofit. All responses are JSON-compatible, and numeric values are returned as numbers (not strings). The Android app can now be a thin client, delegating all calculations to this backend.
